# Issue #32: Docs Refresh

**Link:** https://github.com/thejustinwalsh/three-flatland/issues/32
**Branch (Phase 1):** `docs-refresh-foundation`

## Goal

Modernize the docs site: drop hand-rolled icon and llms.txt hacks in favor of community Starlight plugins, replace the pixel/Jehkoba32 retro aesthetic with a sleek base16-Materia design + new typography stack, and weave three-flatland-powered visualizations into the experience.

## Approach — phased into three PRs

The issue covers four concerns (deps, plugins, typography, theme/redesign). Shipping them in one PR makes it un-reviewable and bisect-hostile, so I'll split:

- **Phase 1 — Foundation** (this PR): dependency upgrades + plugin replacements. Mechanical, easy to review, unblocks the rest.
- **Phase 2 — Typography**: font stack swap (Public Sans / Inter / JetBrains Mono / Commit Mono).
- **Phase 3 — Theme & Design**: base16 Materia palette, minimalist redesign, landing-page interactivity.

Phase 1 ships first because Phase 2/3 land cleanly only after the new plugins (heading-badges, icons) are wired and we know what custom CSS the new Starlight version does/doesn't ship with.

## Phase 1 — scope

### Files likely to change
- `docs/package.json` — bump `astro`, `@astrojs/starlight`, `@astrojs/react`, `starlight-typedoc`, fontsource pkgs; add `starlight-plugin-icons`, `starlight-heading-badges`, `starlight-llms-txt`; remove `pixelarticons` (no longer needed once Icon.astro is gone)
- `docs/astro.config.mjs` — register the three new Starlight plugins; configure llms-txt generation
- `docs/src/components/Icon.astro` — **delete** (replaced by starlight-plugin-icons)
- `docs/src/components/*.astro` — sweep call sites that use `<Icon name="...">` and migrate to the plugin's component / icon names
- `docs/public/llms.txt`, `docs/public/llms-full.txt` — **delete** (plugin generates them at build time)

### Acceptance criteria
- [ ] `pnpm --filter=docs build` (which runs `astro check && astro build`) passes clean
- [ ] Dev server starts and renders without console errors at `/three-flatland/`
- [ ] All previous icon usages render (visual smoke check on landing page, sidebar, code-block tabs)
- [ ] `dist/llms.txt` and `dist/llms-full.txt` are generated by the plugin and are non-empty
- [ ] No leftover references to deleted `Icon.astro` or static `llms*.txt` files

### Risks / open questions
- **Astro 5 → 6 is a major bump.** Latest is `astro@6.2.1` vs current `^5.17.3`. Breaking changes around content layer / config could ripple into the custom Vite plugins (`copy-examples.js`, `watch-examples.js`) and the rehype plugin. Plan: check release notes; if 5.x→6.x is too disruptive for one PR, stay on latest 5.x and file a follow-up issue for the 6.x migration.
- **starlight-typedoc compatibility.** Currently `^0.21.5`; need to confirm it supports the Starlight version we land on. If incompatible, we hold Starlight at the latest version that works.
- **Icon coverage gap.** `starlight-plugin-icons` ships icon sets (Lucide, Heroicons, Tabler, etc.). Some `Icon.astro` mappings (`volume-3`, `volume-x`, `forward-slash`→`animation`, `puzzle`→`human`) are pixelarticons-specific. Need to pick semantic equivalents in the new icon set; document the mapping in `decisions.md`.
- **Sidebar icons.** Issue says "Add icons to the sidebar and codeblocks where applicable." Starlight's sidebar config supports per-item icons via the new plugin — I'll add icons to top-level groups (Getting Started, Guides, Examples, Showcases, API, Project). Codeblock language icons are handled by the plugin's Expressive Code integration if Starlight is using EC.

### Out of scope (Phase 1)
- Typography changes (Phase 2)
- Theme/palette/layout redesign (Phase 3)
- Landing-page interactive scenes (Phase 3)
- Adding heading badges to existing content — plugin gets installed, badge syntax documented, but we don't sweep 311 pages here. Content authors add badges as features ship.
- Removing the user-level `~/.claude/skills/llms-sync` skill — that's a personal-config cleanup, not a repo change. Will note in PR for the maintainer to run separately.

## Phase 2 — Typography (separate PR, planned scope)

- Swap fontsource imports: drop `silkscreen`/`ibm-plex-*`; add `public-sans`, `inter`, `jetbrains-mono`, `commit-mono`.
- Update `retro-theme.css` (likely renamed) typography vars: headings → Public Sans, nav → Inter, prose → JetBrains Mono, code → Commit Mono.
- Verify Commit Mono availability (it's MIT, may not be on Fontsource — fallback: self-host from official site or use JetBrains Mono for code too and revisit).

## Phase 3 — Theme & Design (separate PR, largest scope)

- New `theme.css`: base16 Materia variable mapping → Starlight `--sl-color-*` tokens, light + dark modes.
- Strip pixel-art treatments: `patterns.css` (214 KB of dithered patterns) likely deleted or aggressively trimmed; `RetroBackground`, `RetroHero`, `Hero` re-skinned or replaced.
- Landing-page rebuild: minimalist Ableton/Bitwig-inspired layout with embedded three-flatland scenes.
- Per-page interactive visualizations: select 2-3 high-value guide pages (e.g., TSL Nodes, Pass Effects, Tilemaps) for embedded interactive demos beyond the existing iframe `ExamplePreview`.
- This phase will need its own brainstorming/design pass before implementation — likely coordinate with the `impeccable:frontend-design` skill.
