## Discard — dies with the old aesthetic

### Disposable values
- **Jehkoba32 palette** (`--retro-white/black/pink/purple/cyan/blue/yellow/orange` + dark variants) — replaced by base16 Materia
- **Pixel-art typography** — Silkscreen `--font-pixel`, IBM Plex Sans/Mono — replaced by Public Sans / Inter / JetBrains Mono / Commit Mono
- **Decorative text-shadow on headings** (light + dark) — incompatible with the quiet-layout principle
- **Decorative `border-bottom` on H1/H2** — same; Phase 3 redesign uses spacing + weight, not bordered headings
- **Pattern dither sizes** (`--dither-size`, `--dither-size-2x`) — only used by `patterns.css`, both die together

### Disposable files (whole-file deletion)
- `docs/src/styles/retro-theme.css` (1605 lines)
- `docs/src/styles/patterns.css` (137 lines / ~214 KB of base64-inlined SVG dither patterns)
- `docs/src/styles/global.css` (43 lines — focus-visible and reduced-motion rules migrated to theme's `base.css`; font-family vars die)
- `docs/src/styles/custom.css` (85 lines — sidebar/content-width tokens migrated to theme; search-suggestions CSS migrated to a docs-side stylesheet or override)

### Disposable components
- `docs/src/components/RetroBackground.astro` — pixel-art dither background
- `docs/src/components/RetroHero.astro` — pixel-art hero variant
- `docs/src/components/HeroGradient.astro` — gradient ornament; if needed, the new theme owns hero composition
- The six theme-shaped overrides listed above (Hero, PageFrame, ThemeSelect, SiteTitle, SocialIcons, Head's theme-related parts)

### Disposable upstream CSS-fragment usages
- `pixelarticons` npm dep — already partially replaced by `@iconify-json/pixelarticons` in Phase 1; Phase 2 swaps to `lucide` as default and `tf` custom collection, so even the iconify pixelarticons collection drops out
- `image-rendering: pixelated` / `crisp-edges` rules anywhere — incompatible with the Linear/Radix-leaning aesthetic
- `text-shadow` on any non-decorative element

## Migration order (drives Phase 2.3 → 2.9)

1. **Vendor lucode** into `packages/starlight-theme/` (Phase 2.3)
2. **Replace lucode tokens with base16 Materia** in `theme.css` + `uno.config.ts` (Phase 2.5). Bring forward salvage-list tokens (spacing, sidebar width, content width).
3. **Migrate salvageable rules** to the new theme's `base.css` (focus-visible, reduced-motion).
4. **Wire typography** (Public Sans / Inter / JetBrains Mono / Commit Mono) via `uno.config.ts` `theme.fontFamily` + `starlight-theme/styles/base.css` `@font-face` if needed (Phase 2.6).
5. **Enable `presetWind4` + register `tf` custom icon collection** (Phase 2.7).
6. **Wire the plugin** in `docs/astro.config.mjs`, **compose `SidebarSublist`** in the theme's Sidebar override, **trim docs's `components: { ... }` map** (Phase 2.8).
7. **Migrate site-specific UX** (search suggestions, table-scroll wrapper, optionally GPU detection refactor) to docs-side modules (Phase 2.9).
8. **Delete dead files** — `docs/src/styles/*.css` (all four), the six theme-shaped overrides, RetroBackground, RetroHero, HeroGradient (Phase 2.9).
9. **Verify** — build, agent-browser screenshots in light + dark, `/impeccable:normalize` and `/impeccable:audit` (Phase 2.10–2.11).

The cross-cutting reinit-glue audit in `Head.astro` (theme persist, pagefind reinit, HMR re-dispatch, GPU detection, table-scroll) probes during Phase 2.9. Survivors stay as docs-side glue; refuted items get deleted.
