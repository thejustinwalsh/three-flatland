## What WORKS at the substrate level

1. **Token-driven**: Every `--sl-color-*` Starlight expects, every lucode-derived `--accent`/`--secondary`/`--muted`/`--border`/`--card`/`--popover`/`--ring` token, all flow from a single base16 Materia source in `theme.css`. Both modes coherent.
2. **Plugin composition**: starlight-plugin-icons + starlight-heading-badges + starlight-llms-txt + starlight-typedoc + starlight-theme co-exist; the Sidebar override collision is resolved by reading `data-icon` directly.
3. **Authoring substrate**: `presetWind4` is loaded; utility classes (`bg-bg`, `text-fg`, `font-prose`) read from the same custom-property source. Phase 3 component overrides can author with Tailwind v4–spec utilities.
4. **Aesthetic baseline**: Linear/Radix-leaning ground floor (Design Context principle 2 — "Quiet layout, expressive details"). Generous whitespace, restrained typography, both modes equal first-class citizens.
5. **Accessibility scaffolding**: Reduced-motion + focus-visible + token-driven contrast all in place. Substrate is ready; Phase 3 audit will validate WCAG AA across each redesigned component.
