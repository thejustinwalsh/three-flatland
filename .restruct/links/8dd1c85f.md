## Context Rules
1. `pnpm dev` — docs (port 4000) + examples MPA (port 5174) behind microfrontends proxy at http://localhost:5173
2. `pnpm --filter=example-react-tilemap dev` — run a single example
3. `pnpm sync:pack` — sync example/mini package.json deps with the workspace catalog after editing `pnpm-workspace.yaml`
4. `pnpm sync:react` — regenerate React subpath wrappers after touching `packages/three-flatland/src/*/index.ts`
5. No semicolons, single quotes, trailing commas (Prettier)
6. `type` keyword required for type-only imports (`consistent-type-imports` + `verbatimModuleSyntax`)
7. Unused vars must be prefixed with `_`
8. WebGPU + TSL (Three Shader Language) exclusively — no WebGL, no GLSL
9. R3F examples import from `@react-three/fiber/webgpu`, not `@react-three/fiber`
10. Three.js users: `import from 'three-flatland'` — R3F users: `import from 'three-flatland/react'` (all packages follow this `/react` subpath pattern, incl. `@three-flatland/tweakpane/react`)
11. Shared versions in `pnpm-workspace.yaml` catalog; `pnpm.overrides` maps `@three-flatland/*` to `workspace:*`
12. Examples always exist in **pairs** — Three.js + React. Create both or neither.
13. `examples/three/` = plain Three.js, `examples/react/` = React Three Fiber
14. R3F classes must be registered with `extend()` before use in JSX
15. All Three.js objects used as R3F JSX elements need: optional constructor params, property setters, array-compatible setters

## Constraints
1. Performance is critical — minimize draw calls, batch sprites via SpriteGroup, watch frame budgets
2. All custom Three.js classes must work with R3F's no-arg construction + property-setting pattern

## Workflow
1. Use Conventional Commits — releases are cut from changesets generated from the commit history

## Anti-Patterns
1. Use GLSL or `onBeforeCompile` — all shaders use TSL node materials
2. Use `WebGLRenderTarget` — use renderer-agnostic `RenderTarget`
3. Use Web Awesome (`@awesome.me/webawesome`) — examples use Tweakpane (`@three-flatland/tweakpane/react`) now
4. Add `declare global { namespace JSX }` — use `ThreeElements` interface augmentation via `three-flatland/react`
