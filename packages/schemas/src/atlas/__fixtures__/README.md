# Atlas schema fixtures

Fixtures for `packages/schemas/src/atlas/validator.test.ts` and
`packages/schemas/src/atlas/typeParity.test.ts`. `valid/` fixtures round-trip
through `validateAtlas` with zero errors; `invalid/` fixtures each target one
specific schema violation, asserted by ajv error keyword + instancePath (not
generic pass/fail). `strictFormats.test.ts` (same directory as the schemas,
not under `__fixtures__`) reuses several of these `valid/` fixtures against
the strict per-format `texturepacker.schema.json`/`aseprite.schema.json` —
see that file for what it checks and why.

## Provenance

Where a real committed atlas sidecar in this repo exercises the shape, the
fixture is derived from it (trimmed for brevity) rather than invented, so the
suite stays honest about what real tool output actually looks like. Several
schema fields (multi-format `sources`, `meta.animations`, `meta.normal`,
`mesh`, TexturePacker `vertices`/`verticesUV`/`triangles`) have no real
committed instance yet — those fixtures are synthesized to exercise the
schema branch directly and are labeled as such below.

## Regenerating

These are hand-maintained, not generated. If `knight.json` or the batch-demo
atlas change shape, re-derive the affected fixture by hand and update this
table.
